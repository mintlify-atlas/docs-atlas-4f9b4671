---
title: "GET /api/v1/permission/getByRoleId — Get Permissions for a Role"
sidebarTitle: "Get by Role"
description: "Returns all permissions assigned to a specific role, identified by its numeric roleId query parameter. Requires JWT authentication."
---

Returns all permissions currently assigned to the role identified by `roleId`. The query is performed against the `permissionXRole` pivot table, joining back to the `permissions` table for the full record. If the role has no assigned permissions, an empty array is returned (not a 404).

## Authentication

Requires a valid JWT in the `Authorization` header. No specific role permission is needed.

## Request

**Method:** `GET`
**Path:** `/api/v1/permission/getByRoleId`

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from the login endpoint. Format: `Bearer <token>`
</ParamField>

### Query Parameters

<ParamField query="roleId" type="string" required>
  The numeric ID of the role whose permissions to retrieve. Must match `^\d+$` (digits only, no decimals or negative values).
</ParamField>

## Response

### 200 OK

Returns the list of permissions assigned to the specified role.

<ResponseField name="success" type="boolean">
  Always `true` for successful responses.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable status message.
</ResponseField>

<ResponseField name="data" type="array">
  Array of permission objects assigned to the role. Empty array if the role has no permissions.

  <Expandable title="Permission object">
    <ResponseField name="idPermission" type="number">
      Unique numeric identifier for the permission.
    </ResponseField>

    <ResponseField name="nameUri" type="string">
      The route pattern this permission guards.
    </ResponseField>

    <ResponseField name="description" type="string">
      Human-readable description of the permission.
    </ResponseField>
  </Expandable>
</ResponseField>

```json Example Response
{
  "success": true,
  "message": "Permiso obtenido por rol.",
  "data": [
    {
      "idPermission": 1,
      "nameUri": "GET /api/v1/users",
      "description": "List all users"
    },
    {
      "idPermission": 3,
      "nameUri": "POST /api/v1/roles",
      "description": "Create a new role"
    }
  ]
}
```

```json Empty Role (No Permissions Assigned)
{
  "success": true,
  "message": "Permiso obtenido por rol.",
  "data": []
}
```

### Error Responses

| Status | Description |
|--------|-------------|
| `400 Bad Request` | `roleId` is missing or is not a numeric string. |
| `401 Unauthorized` | Missing or invalid JWT token. |
| `404 Not Found` | Not returned by this endpoint — an unknown `roleId` returns an empty array. |

```json 400 Bad Request
{
  "success": false,
  "message": "Solicitud inválida",
  "data": null
}
```

## Code Example

```bash cURL
curl -X GET \
  -H "Authorization: Bearer <your_token>" \
  "http://localhost:3000/api/v1/permission/getByRoleId?roleId=1"
```

## Socket.IO Events

This is a read-only endpoint. A `data:changed` broadcast is **not** emitted. The requesting client receives `operation:progress` events for the `permissions:fetchByRole` operation on their own socket only.
